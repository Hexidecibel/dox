## Phase 1 — Foundation Fixes

Cleans up the most-painful extraction quality bugs and unifies supplier
matching ahead of the learning-loop work in Phase 2.

### Fixed
- **Filename bleed killed.** The PDF filename moved out of the LLM user
  prompt into a system `<filename_hint>` block with a hard rule against
  extracting structured fields (lot / code / dates / product_code /
  sub_lot) from it. Pacific Cheese was reading lot numbers from
  filenames in text mode — no longer.
- **Compare panel no longer disappears on null noise.** Strip null and
  empty fields from both text and VLM payloads before render and before
  posting to `/api/queue/{id}/results`. Cheese Merchants stayed hidden
  behind a phantom "lots of differences" badge that was actually just
  null-vs-undefined.
- **VLM single-side fallback.** When only VLM extraction exists,
  reviewer state now seeds correctly from the VLM payload instead of
  approving with an empty `primary_metadata`.

### Changed
- **Document type promotion.** `document_type_guess` now promotes to
  the canonical `document_type_id` on exact name or slug match.
  Substring / acronym matches stay as guesses only. Promotions are
  audit-logged for debuggability.
- **Supplier matching unified.** New shared
  `findOrCreateSupplier()` in `functions/lib/suppliers.ts` with
  alias-aware matching replaces the inline lookup logic in
  `queue-approve` and `POST /api/suppliers/lookup-or-create`. New
  supplier-name variants fold into the existing supplier's `aliases`
  array so future writes find the canonical record.
- **VLM diff is now field-level.** Replaced the brittle
  `JSON.stringify` whole-payload diff in `queue-approve` with a
  field-by-field `diffApprovedVsAi` so VLM "Use these results" picks
  reliably trigger an `extraction_examples` write. The insert is now
  guarded when the queue item has no `document_type_id` (NOT NULL
  constraint) and the skip is logged.

### Internal
- Phase 1 of the breezy-hatching-moonbeam plan — sets up the data plane
  for Phase 2's reviewer-decision capture and Phase 3a's learned
  pre-fill.
