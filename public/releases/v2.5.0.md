## Release Versioning System

The app now tells you what version it's running, lets you read the
release notes inline, and pings you when something new ships.

### Added
- **Version chip** in the bottom-right of every page showing
  `v{X}.{Y}.{Z} · {short-sha}`. Click to open the release notes modal.
- **Release notes modal** — MUI dialog with a version dropdown.
  Renders the markdown for any prior release with `react-markdown`,
  styled to match MUI typography.
- **What's-new toast.** On first load after a version bump, a
  dismissible Snackbar surfaces "✨ What's new in v{version} — view
  changes". Click "View" opens the modal; dismissing it persists
  `lastSeenVersion` to localStorage so it doesn't keep nagging.
  Per-user when logged in, falls back to `anon` for the login screen.
  First-ever visit silently marks the current version as seen — no
  toast for new users.
- **`bin/release` script** drafts release notes from `git log
  $LAST_TAG..HEAD`, classifies each commit (Added / Changed / Fixed /
  Internal) by subject, opens the draft in `$EDITOR` for polish,
  bumps `package.json`, commits, tags `vX.Y.Z`, and prompts to deploy.
  Flags: `--patch` (default), `--minor`, `--major`, `--dry-run`.
- **`releases/` directory** with one markdown file per version, plus
  a mirror at `public/releases/` so they're served as static assets.
  An index at `public/releases/index.json` lists every available
  version with date + title.

### Changed
- **`FEATURES.md`** is now a thin index of `releases/v*.md` files —
  detail lives per-release instead of being appended to one ever-growing
  changelog.
- **`package.json`** bumped to `2.5.0` (this PR introduces the system,
  so a minor bump rather than a patch).
- **`vite.config.ts`** injects the short git SHA at build time via
  `define: { __BUILD_SHA__ }` with a try/catch so CI without git falls
  back to `'unknown'`.

### Internal
- New pure modules: `src/lib/versionInfo.ts`, `src/lib/versionCompare.ts`
  (semver compare with no dependency), `bin/lib/classifyCommit.js`
  (classifier for `bin/release`).
- New unit tests: `tests/unit/versionCompare.test.ts`,
  `tests/unit/classifyCommit.test.ts`,
  `tests/unit/releaseNotes.test.ts` (snapshot-style structure check on
  the bootstrap `releases/v2.5.0.md`).
- `tsconfig.json` enables `resolveJsonModule` so the version can be
  imported from `package.json`.
