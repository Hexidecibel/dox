## Phase 3a — Pre-fill, Uncertainty Surfacing, Learning Dashboard

The reviewer-decision capture from Phase 2 closes the loop: workers
pre-fill fields the team has consistently agreed on, the UI surfaces
where it's uncertain, and admins finally see the override-rate trend.

### Added
- **Learned pre-fill.** After VLM canonicalization, the worker fetches
  learned preferences via `/api/learned-preferences` and writes a
  `learned_field_hints` sidecar for fields with confidence ≥0.8. An
  attention-bias line appends to the system prompt when preferences
  exist (hint, not directive).
- **"From N past reviews" chips** next to pre-filled fields in the
  Review Queue. Dismissed-by-learning fields hide behind a "show
  learned-empty" toggle.
- **Per-field uncertainty heuristics.** Empty / short / regex-mismatch
  values raise uncertainty; text–VLM disagreement bumps to ≥0.7. The UI
  sorts uncertain fields to the top with yellow "needs your eyes"
  chips. Confident fields collapse into a "Looks good (N fields)"
  accordion.
- **Override-rate dashboard** at `/admin/learning-dashboard` with four
  panels: override-rate trend, review-burden trend, trust-distribution
  placeholder (Phase 3b), top suppliers by pick volume. Inline SVG
  charts — no chart library was added.
- **Backfill on approve.** When `(supplier, doctype)` rolls up to
  high-confidence learned state and `pick_count` crosses a multiple of
  5, `queue-approve` synthesizes a canonical `extraction_examples` row
  using `input_text_hash` plus the partial unique index from migration
  0038. Feeds the few-shot loop without requiring real text input.
- **43 new tests** — 17 aggregator, 17 uncertainty heuristics, 9 sort.

### Changed
- **Migration 0039** adds `learned_field_hints` plus uncertainty
  columns to `processing_queue` (both nullable JSON). Workers populate;
  the UI consumes.
- **`learnedPreferences` aggregator** is a pure function over
  `reviewer_field_picks` / `dismissals` / `table_edits`. Excludes tied
  picks (text === VLM) from the source-preference vote so the
  alphabetical text-tie-break doesn't skew results. Requires ≥3
  non-tied picks before exposing a preference.

### Internal
- Phase 3a of the breezy-hatching-moonbeam plan. Phase 3b (trust
  ladder) is intentionally deferred until 1+ week of override-rate data
  shows the loop is working.
