## The Records module ships

A collaborative tracker module modeled after Smartsheet but built around the design philosophy that **the record is the truth, the grid is just one view**. Mobile-first throughout — built to be the wedge against tools that treat phones as an afterthought.

## What's new

### Records — collaborative trackers
- 5 view types: Grid, Kanban, Calendar, Timeline, Gallery (switchable in one click)
- Native entity references: supplier / product / customer / record / contact / document chips that hover-preview and click-navigate
- Real-time presence + cell updates via Cloudflare Durable Objects
- Rich row drawer with activity feed (actor + before-after attribution)
- Manage columns from any view (sheet settings menu) — edit dropdown options on mobile

### Forms
- Public-link intake forms derived from a sheet's column schema
- Typeform-feel mobile renderer with Cloudflare Turnstile
- Photo / file attachment uploads with mobile camera capture
- Tenant-scoped entity dropdowns (customers / suppliers / products)
- Form builder: split-pane editor with live preview and status workflow

### Update Requests
- Send a targeted form to a specific person (user OR raw email) asking them to fill specific fields on a specific row
- Magic-link recipient form at `/u/:token` with diff-style review

### Workflows
- Linear step-by-step workflow builder per sheet
- Three step types: approve/reject, send Update Request, set cell value
- Branching via on_approve_next / on_reject_next
- Approvers: in-app inbox at `/approvals` OR magic link at `/a/:token`
- Run visualization with avatars + live status (the demo artifact)

### Mobile-first throughout
- Stacked cards instead of grids on phones; full-screen modals instead of side drawers; bottom sheets instead of popovers; thumb-zone action bars

## Infrastructure
- New Durable Object Worker (`dox-sheet-session`) bound by Pages via `script_name`
- Migrations 0040-0045: 14 new tables for the Records module
- `bin/migrate` now tracks applied migrations via `_migrations` table
- `bin/add-pages-domain`, `bin/set-pages-secret` reusable Cloudflare scripts
- Cush-tools 1Password integration for `.dev.vars*`
